# Meta App権限設定 - 詳細手順

## 現在の状況

表示されている権限（email, user_birthday等）は**個人情報**に関するもので、今回は不要です。

必要なのは**ページとInstagramのビジネスデータ**にアクセスする権限です。

---

## 正しい権限を追加する手順

### ステップ1: ダッシュボードに戻る

1. Meta for Developersで、左上の「マイアプリ」または「ダッシュボード」をクリック
2. あなたのアプリ（App ID: 1606160517196384）のダッシュボードに戻る

### ステップ2: 製品セクションを確認

左サイドバーで以下を確認:

```
📱 ダッシュボード
⚙️ 設定
   └ ベーシック
   └ 詳細設定
📦 製品
   └ Instagram (あるか確認)
   └ Facebookログイン (あるか確認)
📋 アプリレビュー
🔧 ツール
```

### ステップ3: Facebookログイン製品を追加

**もし左サイドバーに「Facebookログイン」が表示されていない場合:**

1. ダッシュボードのメインエリアで「製品を追加」セクションを探す
2. 「Facebookログイン」を見つける
3. 「設定」ボタンをクリック
4. 設定ウィザードが表示されたら「ウェブ」を選択
5. サイトURL: `http://localhost:3000`を入力
6. 「保存」をクリック

**もし左サイドバーに「Facebookログイン」が既に表示されている場合:**

1. 左サイドバーで「Facebookログイン」をクリック
2. 「設定」をクリック
3. 「有効なOAuthリダイレクトURI」に以下を追加:
   ```
   http://localhost:3000/auth/meta/callback
   ```
4. 「変更を保存」をクリック

### ステップ4: Instagram製品を確認

**もし左サイドバーに「Instagram」が表示されていない場合:**

1. ダッシュボードのメインエリアで「製品を追加」セクションを探す
2. 「Instagram」を見つける
3. 「設定」ボタンをクリック

**もし左サイドバーに「Instagram」が既に表示されている場合:**

1. 左サイドバーで「Instagram」をクリック
2. サブメニューを確認（「基本表示」「ビジネスログイン」等）

### ステップ5: アプリレビューで権限を確認

1. 左サイドバーで「アプリレビュー」をクリック
2. 以下のタブを探す:
   - 「権限とフィーチャー」
   - 「リクエスト」
   - 「アクセス許可」
3. 検索ボックスで以下を検索:
   - `pages_show_list`
   - `pages_read_engagement`
   - `instagram_basic`
   - `instagram_manage_insights`

## 重要: 開発モードでは権限は自動付与される

実は、**開発モード**では多くの権限が自動的に利用可能になります。

### 確認方法

1. 「設定」→「ベーシック」をクリック
2. ページ上部の「アプリモード」を確認
3. **「開発モード」**になっていれば、権限は自動的に付与されている可能性が高い

## 問題の本質: ページが取得できていない

サーバーログ: `Total pages found: 0`

これは権限の問題ではなく、以下の可能性があります:

### 可能性1: Instagramアカウントがページにリンクされていない

**確認方法:**

1. Facebookで「BuzzInsight」ページを開く
2. 左サイドバーの「設定」をクリック
3. 「Instagram」セクションを探す
4. Instagramアカウントがリンクされているか確認

**もしリンクされていない場合:**

1. 「アカウントをリンク」をクリック
2. Instagramにログイン
3. ビジネスアカウントに切り替え（まだの場合）
4. リンクを承認

### 可能性2: 間違ったFacebookアカウントでログインしている

**確認方法:**

接続テスト時に、**BuzzInsightページを作成したFacebookアカウント**でログインしていますか？

別のアカウントでログインすると、ページが見つかりません。

### 可能性3: ページの管理者権限がない

**確認方法:**

1. Facebookで「BuzzInsight」ページを開く
2. 左サイドバーの「設定」→「ページの役割」をクリック
3. あなたのアカウントが「管理者」になっているか確認

## 次のステップ: 直接テストする

権限設定を気にせず、まず接続テストを実行してみましょう。

### 1. サーバーを再起動

```powershell
# サーバーのPowerShellでCtrl+Cを押して停止
npm run dev
```

### 2. ブラウザのキャッシュをクリア

- Ctrl+Shift+Delete
- 「Cookieと他のサイトデータ」と「キャッシュされた画像とファイル」をチェック
- 「データを削除」をクリック

### 3. 接続テスト

1. `http://localhost:3000/settings`にアクセス
2. 「Instagram」の「接続する」ボタンをクリック
3. ポップアップが開く
4. **BuzzInsightページを作成したFacebookアカウント**でログイン
5. 権限を承認

### 4. サーバーログを確認

PowerShellのサーバーログで以下を確認:

```
=== Meta API Response ===
Total pages found: ?  ← この数字を確認
========================
```

- **0の場合**: Instagramがページにリンクされていない
- **1以上の場合**: 成功！

## トラブルシューティング: Total pages found: 0

もし`Total pages found: 0`が続く場合:

### 解決策1: Instagramを再リンク

1. Facebookで「BuzzInsight」ページを開く
2. 「設定」→「Instagram」
3. 既にリンクされている場合は「リンクを解除」
4. 再度「アカウントをリンク」をクリック
5. Instagramにログイン
6. ビジネスアカウントであることを確認
7. リンクを承認

### 解決策2: 別のページを作成

もし上記で解決しない場合、新しいFacebookページを作成:

1. Facebookで「ページ」→「新しいページを作成」
2. ページ名: `BuzzInsight Test`
3. カテゴリ: `ビジネス・経済`
4. 「ページを作成」をクリック
5. 「設定」→「Instagram」でアカウントをリンク
6. 再度接続テスト

### 解決策3: Graph API Explorerで直接確認

1. [Graph API Explorer](https://developers.facebook.com/tools/explorer/)にアクセス
2. 右上で「Meta App」を選択（あなたのアプリ: 1606160517196384）
3. 「アクセストークンを取得」をクリック
4. 以下の権限を選択:
   - pages_show_list
   - pages_read_engagement
   - instagram_basic
5. 「アクセストークンを生成」をクリック
6. クエリフィールドに以下を入力:
   ```
   me/accounts?fields=id,name,instagram_business_account
   ```
7. 「送信」をクリック
8. レスポンスを確認:
   - ページが表示される → 権限OK、リンクの問題
   - エラーが表示される → 権限の問題

## まとめ

1. **権限設定は後回しでOK** - 開発モードでは自動付与される
2. **まず接続テストを実行** - 実際の動作を確認
3. **サーバーログで`Total pages found`を確認** - 0なら原因を特定
4. **Instagramのリンク状態を確認** - これが最も重要
